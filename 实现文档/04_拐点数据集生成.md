# 拐点数据集生成

## 1. 目标

本文件说明如何生成拐点数据集。拐点数据集主要服务任务 2，也可以作为任务 1 的辅助分析材料。

拐点定义：

| 类型 | 含义 |
| --- | --- |
| `1` | 峰型拐点，前面上升、后面下降 |
| `-1` | 谷型拐点，前面下降、后面上升 |
| `0` | 非拐点 |

## 2. 输入文件

```text
ETTh1.csv
```

目标列：

```text
HUFL, HULL, MUFL, MULL, LUFL, LULL, OT
```

现有脚本可复用：

```text
自适应显著性拐点检测.py
```

该脚本已经包含平滑、峰谷候选点检测、显著性过滤和结果输出逻辑。

## 3. 基本生成流程

拐点数据集生成建议按以下步骤：

```text
1. 读取 ETTh1.csv
2. 对每个目标列转成数值序列
3. 对序列做平滑，降低小时级噪声
4. 在平滑序列上寻找峰型和谷型候选点
5. 用滚动 MAD 计算自适应显著性阈值
6. 过滤显著性不足的候选点
7. 将候选点回贴到原始序列中的真实局部极值位置
8. 输出每小时一行的拐点标签
```

## 4. 平滑处理

建议使用 Savitzky-Golay 平滑。

作用：

```text
减少微小抖动
保留局部形状
让峰谷检测更稳定
```

默认参数可沿用现有脚本：

```text
平滑窗口 = 5
多项式阶数 = 2
```

如果拐点过多，说明平滑不足或显著性阈值过低。

## 5. 峰谷候选点检测

在平滑序列上分别检测：

```text
峰型候选点：局部高点
谷型候选点：局部低点
```

候选点不是最终标签，还需要经过显著性过滤和原始序列回贴。

可使用的判据：

```text
局部极值关系
prominence 显著性
minimum_distance 最小间隔
minimum_width 最小宽度
```

## 6. 自适应显著性阈值

不同时间段波动幅度不同，固定阈值容易失效。因此使用滚动 MAD 估计局部波动强度。

MAD 是中位数绝对偏差，可理解为一种稳健波动度量。

默认：

```text
MAD 窗口 = 168 小时
显著性倍数 = 0.5
```

候选点显著性必须大于当前时间附近的自适应阈值，才保留为有效拐点。

## 7. 回贴原始序列

平滑会让峰值位置轻微偏移。为了让拐点时间更贴近真实数据，应将平滑序列中的候选点回贴到原始序列附近窗口内的真实局部极值位置。

例如：

```text
平滑序列检测到 t=08:00 是峰
在原始序列 t 附近寻找真实最大值
将拐点标记放到真实最大值所在小时
```

这样可以避免标签和原始数据峰值错位。

## 8. 输出字段

每个变量单独输出一个拐点文件，建议字段：

```text
date
value
smooth_value
turning_type
turning_prominence
turning_threshold
valid_mask
```

字段含义：

| 字段 | 含义 |
| --- | --- |
| `date` | 当前小时 |
| `value` | 原始值 |
| `smooth_value` | 平滑后的值 |
| `turning_type` | 拐点类型，`1`、`-1`、`0` |
| `turning_prominence` | 当前拐点显著性 |
| `turning_threshold` | 当前自适应阈值 |
| `valid_mask` | 当前点是否有效，缺失或边界不可判定时为 0 |

建议输出：

```text
代码实现/数据集/拐点标签/HUFL拐点.csv
代码实现/数据集/拐点标签/HULL拐点.csv
代码实现/数据集/拐点标签/MUFL拐点.csv
代码实现/数据集/拐点标签/MULL拐点.csv
代码实现/数据集/拐点标签/LUFL拐点.csv
代码实现/数据集/拐点标签/LULL拐点.csv
代码实现/数据集/拐点标签/OT拐点.csv
```

## 9. 与任务 1 的关系

拐点数据集可用于：

```text
分析峰值附近是否存在显著拐点
辅助解释 peak_hour 预测失败原因
后续作为模型后处理或风险提示
第二阶段任务 2 的监督标签
```

但要注意，任务 1 第一版不把未来真实拐点作为输入特征。否则会产生信息泄露。

可以使用的安全方式：

```text
只使用预测起点之前的历史拐点统计
例如过去 96 小时拐点数量、峰型拐点数量、谷型拐点数量、最近拐点距离
```

不能使用：

```text
未来 14 天真实拐点
由未来窗口计算出的拐点显著性
未来 peak_hour 附近是否为拐点
```

## 10. 质量检查

生成拐点数据集后应检查：

| 检查项 | 要求 |
| --- | --- |
| 行数 | 与原始小时级数据一致 |
| 时间列 | 与原始 `date` 对齐 |
| 类型取值 | 只能为 `-1`、`0`、`1` |
| 有效标记 | 边界和缺失处理清晰 |
| 拐点密度 | 不能过密，也不能几乎没有 |
| 可视化抽查 | 随机选择若干时间段画图验证 |
