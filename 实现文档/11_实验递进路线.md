# 实验递进路线

## 1. 总体原则

实验递进遵循：

```text
先简单、可解释、可对照
再增加历史长度
再缩短预测长度
最后加入更复杂的小波特征
```

不要在第一版同时改变多个因素，否则无法判断是哪一项带来提升或退化。

## 2. 第一阶段：默认实验

配置：

```text
seq_len = 96h
pred_len = 336h
特征 = 统计特征
模型 = LightGBM + XGBoost
小波 = 不使用
```

实验：

```text
E00 规则基线
E01 LightGBM
E02 XGBoost
```

通过条件：

```text
至少一个模型在 peak_value MAE 上优于最佳规则基线
多数变量有提升
预测分布没有明显塌缩
```

## 3. 第二阶段：增长输入长度

如果第一阶段效果不好，先保持：

```text
pred_len = 336h
```

依次测试：

```text
seq_len = 192h
seq_len = 336h
seq_len = 672h
```

目的：

```text
判断过去 4 天是否信息不足
观察更长历史是否改善未来 14 天峰值预测
```

停止条件：

```text
连续两个更长 seq_len 没有提升
或训练集提升但测试集下降
```

## 4. 第三阶段：缩短预测长度

如果增长输入后仍无法预测未来 14 天，则固定最佳 `seq_len`，缩短 `pred_len`：

```text
pred_len = 192h，即未来 8 天
pred_len = 96h，即未来 4 天
```

目的：

```text
判断任务是否只在短中期可预测
避免远期 horizon 拖垮整体指标
```

如果短期明显好、远期接近基线，说明未来 14 天峰值本身可能不可稳定预测。

## 5. 第四阶段：加入小波特征

在第一版主流程稳定后加入小波特征。

配置：

```text
特征 = 统计特征 + 小波特征
模型 = LightGBM + XGBoost
```

与无小波版本做消融比较。

通过条件：

```text
测试集 peak_value MAE 下降
远 horizon 或极端峰值预测改善
模型没有明显过拟合
```

## 6. 第五阶段：拆分模型粒度

如果发现某些 horizon 表现差异很大，可从：

```text
每列一个模型，horizon 作为特征
```

改为：

```text
每列 × 每个 horizon 单独一个模型
```

优点：

```text
每个 horizon 单独学习
更容易诊断远期和近期差异
```

缺点：

```text
模型数量增加
单个模型样本更少
维护成本更高
```

## 7. 每阶段记录要求

每个阶段必须记录：

```text
实验编号
seq_len
pred_len
特征版本
模型类型
训练样本数
验证样本数
测试样本数
最佳基线
模型评估结果
是否进入下一阶段
原因
```

结果摘要写入 [00_总文档.md](00_总文档.md)，详细结果写入 [09_评估指标与结果记录.md](09_评估指标与结果记录.md) 指定结果文件。
