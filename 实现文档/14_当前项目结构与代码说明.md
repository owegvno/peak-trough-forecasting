# 当前项目结构与代码说明

本文记录当前项目的目录结构、目录用途、主要文件职责，以及 Python 文件中各函数的输入、输出和作用。

当前项目根目录为：

```text
/home/wmb/test/波峰波谷预测
```

## 1. 项目整体理解

本项目基于 `ETTh1.csv` 小时级时间序列数据，围绕 7 个变量：

```text
HUFL, HULL, MUFL, MULL, LUFL, LULL, OT
```

构造以自然日为单位的波峰、波谷预测实验数据。当前默认实验配置为：

```text
seq_days = 4，即过去 96 小时作为输入窗口
pred_days = 14，即未来 336 小时作为预测窗口
周期单位 = 完整自然日 00:00-23:00
```

当前主数据流为：

```text
ETTh1.csv
  -> src/wave_dataset/build_datasets.py
  -> 数据集/ETTH1_pred14_seq4/
     -> 每变量宽表样本
     -> 每变量拐点标签
  -> src/wave_dataset/long_table.py
  -> 数据集/ETTH1_pred14_seq4/长表/
  -> src/wave_experiments/datasets/validate_peak_dataset.py
  -> src/wave_experiments/baselines/
     -> peak_value 规则基线
     -> peak_hour 规则基线
     -> 验证集最佳基线选择
  -> 实验输出/
     -> 验证报告
     -> 基线预测与指标
     -> 残差学习数据
  -> 数据集可视化/
     -> 样本标签图
     -> 规则基线波峰预测图
```

当前已经实现的阶段包括：

- 峰值、谷值样本数据集生成。
- 拐点标签生成。
- 宽表到长表转换。
- 长表数据集验证。
- `peak_value` 多种规则基线。
- `peak_hour` 多种规则基线。
- 按验证集选择每个 `目标变量 + 预测天数` 的最佳规则基线。
- 基于最佳规则基线生成残差学习数据。
- 数据集与规则基线可视化。

## 6. 当前数据与产物关系

### 6.1 宽表

宽表位于：

```text
数据集/ETTH1_pred14_seq4/<变量>/
```

一行代表一个预测起点和一个目标变量，包含：

- 样本 ID。
- 预测起点日期。
- 数据集划分。
- 未来 14 天的目标峰值、目标峰值残差、目标峰值小时、目标谷值、目标谷值残差、目标谷值小时。
- 当前目标变量过去 4 天和过去 96 小时统计特征。

宽表适合人工检查和按变量查看，但不如长表适合统一模型训练。

### 6.2 长表

长表位于：

```text
数据集/ETTH1_pred14_seq4/长表/
```

一行代表：

```text
样本ID + 目标变量 + 预测天数
```

长表将宽表中的 `第1天_目标峰值` 到 `第14天_目标峰值` 拆为 14 行，并合并全部 7 个变量的历史特征。规则基线、数据验证和后续模型训练都主要使用长表。

### 6.3 拐点标签

拐点标签位于：

```text
数据集/ETTH1_pred14_seq4/拐点标签/
```

每个变量一个 CSV，每行对应原始数据中的一个小时，主要字段包括：

- 日期。
- 原始值。
- 平滑值。
- 拐点类型：`1=峰型`、`-1=谷型`、`0=非拐点`。
- 拐点显著性。
- 拐点阈值。
- 有效标记。

当前拐点标签作为任务 2 标签和辅助分析材料，不默认并入波峰预测样本。

### 6.4 规则基线产物

规则基线输出位于：

```text
实验输出/results/baselines/
```

其中 `peak_value` 规则包括：

- `mean_last_4`：过去 4 天峰值均值。
- `weighted_mean_last_4`：过去 4 天峰值加权均值。
- `cycle_mod_4`：按 horizon 复用过去 4 天对应位置峰值。
- `weekday_horizon_mean`：训练集内按目标变量、星期、horizon 统计均值，并带回退。

`peak_hour` 规则包括：

- `mode_last_4`：过去 4 天峰值小时众数。
- `median_last_4`：过去 4 天峰值小时中位数。
- `global_mode`：训练集内目标变量峰值小时众数，缺失时回退全局众数。
- `weekday_mode`：训练集内目标变量+星期峰值小时众数，缺失时回退。

最佳基线选择只使用验证集 `target_variable_horizon` 层级：

- `peak_value` 按 MAE 最低选择。
- `peak_hour` 按 ±2h 命中率最高选择，并列时普通小时误差最低。

## 8. 维护注意事项

1. 字段名同时存在内部英文概念和输出中文字段，生产 CSV 时主要使用中文字段。新增字段时应优先更新 `config.py` 中的映射和基础字段列表。

2. 宽表和长表用途不同。宽表适合按变量检查样本，长表适合规则基线和模型训练。后续模型优先读取 `长表/峰谷预测长表_seq96_pred336_全部变量.csv`。

3. 时间切分由预测窗口决定，跨训练/验证/测试边界的样本会被丢弃。不要只按预测起点判断 split。

4. `peak_value` 的训练集统计基线只能使用训练集统计量；验证集和测试集真实目标不能参与统计。

5. `peak_hour` 是 0-23 的环形小时任务，评估时普通小时误差和环形小时误差含义不同。

6. 最佳规则基线选择只使用验证集指标。测试集只用于最终泛化评估，不参与选择。

7. `validate_peak_dataset.py` 的泄露检查基于字段名关键词，新增特征时应避免使用 `target`、`future`、`label`、`目标` 等容易被误判为泄露的命名，除非该字段确实不是模型输入。

8. `turning_points.py` 生成的拐点标签当前是辅助数据，不会自动进入长表特征。若后续要加入模型，需要新增明确的特征工程步骤和泄露检查。

9. `自适应显著性拐点检测.py` 和 `src/wave_dataset/turning_points.py` 思路相近，但前者是独立中文脚本，后者是当前数据集生成流程的一部分。维护时应注意两者参数和输出字段不完全相同。

10. `数据集/`、`数据集可视化/`、`实验输出/` 中多数文件是生成产物，代码逻辑应以 `src/` 和 `tests/` 为准。
