## 2. 已确认实验目标

本实验目标是处理 `ETTh1.csv` 数据集，以自然日 `00:00-23:00` 为一个周期，预测未来多个自然日内 7 个变量的每日波峰。

目标变量：

```text
HUFL, HULL, MUFL, MULL, LUFL, LULL, OT
```

每个变量每天预测两个目标：

```text
peak_value：当天最大值
peak_hour：当天最大值第一次出现的小时，范围 0-23
```

第一版默认实验配置：

```text
seq_len = 96 小时，即过去 4 天
pred_len = 336 小时，即未来 14 天
周期 = 固定自然日 00:00-23:00
输入 = 7 列历史数据共同作为特征
输出 = 每列单独建模
优先级 = peak_value 和 peak_hour 都重要，但冲突时优先保证 peak_value
```

## 3. 已确认关键决策

| 决策项 | 当前决定 |
| --- | --- |
| 周期切分 | 固定自然日周期，`00:00-23:00` |
| 不完整日期 | 直接丢弃，不作为输入日，也不作为预测目标日 |
| 重复最大值 | 取第一次出现的小时作为 `peak_hour` |
| 数据切分 | 使用 ETTH1 常见 `12个月 / 4个月 / 4个月` 时间切分 |
| 输入变量 | 每个目标都使用 7 列历史作为输入 |
| 输出方式 | 按变量分开建模 |
| 第一版方法 | 规则基线加残差学习，加 LightGBM 和 XGBoost |
| 第一版是否用小波 | 不使用 |
| 第二版方法 | 在第一版基础上加入小波特征 |
| 效果不好时 | 先增加 `seq_len`，再缩短 `pred_len`，最后加入或扩展小波特征 |

## 6. 如何判断模型是否真正学到东西

模型不能只看训练集误差。首要判断标准是测试集是否稳定超过规则基线。

一个模型通过第一版验证，需要尽量满足：

| 判断项 | 通过标准 |
| --- | --- |
| peak_value 测试集误差 | 优于最佳规则基线 |
| 相对提升 | MAE 稳定降低 5% 以上更有价值 |
| 变量覆盖 | 多数变量提升，而不是只靠单个变量拉低平均误差 |
| horizon 覆盖 | 多数未来天数提升，而不是只在第 1 天有效 |
| 残差信号 | 预测残差与真实残差有正相关 |
| 分布检查 | 预测峰值分布不能明显塌缩 |
| peak_hour | 至少在 ±2h 命中率上优于时间基线 |

如果 `peak_value` 明显提升但 `peak_hour` 一般，模型仍可保留。若 `peak_hour` 提升但 `peak_value` 变差，不作为主模型。
