# 规则基线与残差学习

## 1. 目标

规则基线用于回答一个核心问题：

```text
机器学习模型是否真的比简单规则更好
```

残差学习用于让模型学习相对基线的修正量，而不是直接预测绝对峰值。

## 2. 为什么必须先做规则基线

如果不做规则基线，模型误差看起来较低也不能说明它有效。因为每日峰值可能本身有强均值规律，简单规则就能达到不错效果。

因此第一步必须建立强基线，然后所有模型都与最佳基线比较。

## 3. peak_value 规则基线

### 3.1 过去 4 天峰值均值

```text
baseline_mean = mean(过去 4 天 peak_value)
```

未来 14 天都使用同一个 `baseline_mean`。

### 3.2 过去 4 天峰值加权均值

越近日期权重越高，例如：

```text
权重 = [0.1, 0.2, 0.3, 0.4]
baseline_weighted = sum(weight_i * peak_value_i)
```

### 3.3 h mod 4 对应历史日峰值

未来第 `h` 天使用过去 4 天中相同循环位置的峰值。

示例：

```text
horizon=1 使用最近第 1 天
horizon=2 使用最近第 2 天
horizon=5 再次使用最近第 1 天
```

### 3.4 weekday/horizon 均值

在训练集中统计：

```text
目标变量 + weekday + horizon 的平均 peak_value
```

预测时根据未来目标日的星期和 horizon 查表。

该基线可以衡量日历规律是否强于短期历史规律。

## 4. peak_hour 规则基线

候选基线：

```text
过去 4 天 peak_hour 众数
过去 4 天 peak_hour 中位数
训练集该变量最常见 peak_hour
训练集该变量在相同 weekday 最常见 peak_hour
```

如果 `peak_hour` 模型不能超过这些基线，说明峰值时间可能难以预测，或特征不足。

## 5. 残差学习定义

对每个样本、目标变量、horizon：

```text
target_peak_residual = target_peak_value - baseline_peak
```

模型训练目标为 `target_peak_residual`。

预测时：

```text
pred_peak_value = baseline_peak + pred_peak_residual
```

## 6. 为什么残差学习适合本实验

7 个变量尺度差异明显。例如 `OT` 的数值范围和波动幅度通常大于 `LULL`。直接预测绝对峰值时，模型更容易被大尺度变量主导。

残差学习的优点：

```text
降低尺度差异影响
让模型专注学习偏离正常水平的部分
更容易判断模型是否有修正能力
可以保留规则基线的稳定性
```

## 7. 如何判断模型是否真正学到东西

### 7.1 与最佳基线比较

计算：

```text
improvement = (baseline_MAE - model_MAE) / baseline_MAE
```

若测试集 MAE 稳定下降，说明模型有价值。

### 7.2 残差相关性

检查：

```text
corr(pred_residual, true_residual)
```

如果相关性接近 0，模型基本没有学到修正方向。

### 7.3 残差模型是否优于预测 0

预测 0 表示完全相信基线。

如果模型预测残差后反而更差，则模型没有增加价值。

### 7.4 预测分布检查

检查真实峰值和预测峰值的：

```text
均值
标准差
最大值
分位数
```

如果预测分布明显变窄，说明模型仍然在向均值塌缩。

## 8. 第一版推荐执行顺序

```text
1. 生成日峰值标签
2. 生成规则基线预测
3. 评估规则基线
4. 选择每列每个 horizon 的最佳基线
5. 构造 residual 标签
6. 训练 LightGBM 和 XGBoost
7. 将预测残差加回基线
8. 与最佳基线比较
```
