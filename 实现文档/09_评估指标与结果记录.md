# 评估指标与结果记录

## 1. 目标

本文件定义如何评估规则基线、LightGBM、XGBoost 以及后续小波增强模型。

评估必须回答：

```text
模型是否超过最佳规则基线
模型在哪些变量上有效
模型在哪些 horizon 上有效
模型是否真的预测了峰值，而不是输出均值
```

## 2. peak_value 指标

### 2.1 MAE

平均绝对误差：

```text
MAE = mean(abs(pred_peak_value - target_peak_value))
```

第一优先级指标。

### 2.2 RMSE

均方根误差：

```text
RMSE = sqrt(mean((pred_peak_value - target_peak_value)^2))
```

对大误差更敏感，用于判断极端峰值是否预测失败。

### 2.3 sMAPE

对称平均绝对百分比误差：

```text
sMAPE = mean(2 * abs(pred - true) / (abs(pred) + abs(true)))
```

用于跨变量比较，但当真实值接近 0 时要谨慎解释。

## 3. peak_hour 指标

### 3.1 平均小时误差

```text
hour_error = abs(pred_peak_hour - target_peak_hour)
```

由于一天有 24 小时，也可以考虑环形距离：

```text
circular_error = min(abs(a-b), 24-abs(a-b))
```

第一版建议同时记录普通距离和环形距离。

### 3.2 Top-1 accuracy

预测小时完全等于真实小时的比例。

### 3.3 ±1h 命中率

```text
abs(pred_peak_hour - target_peak_hour) <= 1
```

### 3.4 ±2h 命中率

```text
abs(pred_peak_hour - target_peak_hour) <= 2
```

`±2h` 命中率是更实际的辅助指标。

## 4. 分层评估

每次实验必须至少生成以下评估表。

### 4.1 全局平均

```text
所有变量 × 所有 horizon 的平均指标
```

### 4.2 按变量

分别评估：

```text
HUFL, HULL, MUFL, MULL, LUFL, LULL, OT
```

### 4.3 按 horizon

分别评估：

```text
horizon = 1, 2, ..., 14
```

### 4.4 按变量加 horizon

最细粒度：

```text
变量 × horizon
```

用于定位某个变量或某个未来天数表现很差。

## 5. 与基线比较

计算相对提升：

```text
improvement = (baseline_MAE - model_MAE) / baseline_MAE
```

解释：

| improvement | 含义 |
| --- | --- |
| 大于 0 | 模型优于基线 |
| 等于 0 | 模型与基线持平 |
| 小于 0 | 模型不如基线 |
| 大于 5% | 初步认为有明确价值 |

## 6. 残差检查

对残差模型检查：

```text
corr(pred_residual, true_residual)
MAE(pred_residual, true_residual)
MAE(0, true_residual)
```

如果预测残差不如直接预测 0，说明模型没有提供修正价值。

## 7. 分布检查

比较真实峰值和预测峰值：

```text
均值
标准差
最小值
最大值
P10
P50
P90
P95
```

如果预测标准差明显小于真实标准差，说明模型可能发生峰值塌缩。

## 10. 失败判定

以下情况说明实验效果不好：

```text
peak_value MAE 不如最佳规则基线
多数变量没有提升
horizon 1-3 之外全部退化
预测峰值分布明显塌缩
预测残差和真实残差相关性接近 0
peak_hour 不如简单时间基线
```

如果第一版失败，按 [11_实验递进路线.md](11_实验递进路线.md) 调整。
