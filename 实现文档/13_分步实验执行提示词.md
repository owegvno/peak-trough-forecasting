# 分步实验执行提示词

## 1. 文档目的

本文档把 ETTH1 周期波峰预测实验拆成可逐步执行的小任务。每一步都包含：

```text
1. 本步目标
2. 完整提示词
3. 验收标准
```

使用方式：

```text
每次只执行一步。
执行完成后按验收标准检查。
通过后再进入下一步。
```

## 2. 全局执行规则

所有步骤默认遵守以下规则。

| 项目 | 约定 |
| --- | --- |
| 工作目录 | `/home/wmb/test/wmb/ETT数据预处理` |
| Windows 访问路径 | `\\wsl.localhost\Ubuntu-24.04\home\wmb\test\wmb\ETT数据预处理` |
| 代码目录 | `代码实现/` |
| 文档目录 | `实现文档/` |
| 运行环境 | WSL 中的 conda 环境 `Ada-MSHyper` |
| 运行方式 | `conda run -n Ada-MSHyper python 脚本路径.py` |
| 原始数据 | `ETTh1.csv` |
| 目标变量 | `HUFL,HULL,MUFL,MULL,LUFL,LULL,OT` |
| 第一版输入长度 | `seq_len=96h`，过去 4 天 |
| 第一版预测长度 | `pred_len=336h`，未来 14 天 |
| 第一版模型 | LightGBM 和 XGBoost |
| 第一版小波 | 不使用 |

所有实现应尽量放在：

```text
代码实现/
```

建议目录结构：

```text
代码实现/
  datasets/
  baselines/
  features/
  models/
  evaluation/
  results/
  reports/
```

如已有数据集生成代码，应优先复用，不要重复实现同一逻辑。

## 3. 总体执行顺序

| 阶段 | 步骤 | 内容 |
| --- | --- | --- |
| A | 第 1-2 步 | 检查已生成数据集，生成验证报告 |
| B | 第 3-6 步 | 实现并评估规则基线 |
| C | 第 7-8 步 | 生成残差标签和统一特征表 |
| D | 第 9-10 步 | 实现 LightGBM 第一版 |
| E | 第 11-12 步 | 实现 XGBoost 第一版 |
| F | 第 13-14 步 | 汇总结果并判断模型是否学到东西 |
| G | 第 15-16 步 | 做 `seq_len` 和 `pred_len` 递进实验 |
| H | 第 17-18 步 | 做第二版小波增强实验 |

### 完整提示词

```text
1. 找到已生成的数据集文件，并列出文件路径。
```


## 4. 总结

执行时建议严格按顺序推进：

```text
1-2：先确认数据集可靠
3-6：先建立规则基线
7-8：再生成残差目标和模型矩阵
9-12：训练 LightGBM 与 XGBoost
13-14：汇总和诊断
15-16：根据结果调整窗口
17-18：最后加入小波特征
```

每一步通过验收后，再进入下一步。若某一步验收失败，应先修复当前步骤，不要继续堆叠后续模型。
