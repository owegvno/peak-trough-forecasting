# 方法原理与适用范围

## 1. 统计方法

统计方法是指通过数学统计量描述时间序列的水平、波动、趋势、周期和异常程度。常见统计量包括均值、标准差、最大值、最小值、分位数、滑动窗口统计、差分、相关性、峰谷差等。

### 1.1 原理

统计方法不依赖复杂模型，而是直接从历史数据中计算可解释特征。例如：

```text
过去 4 天最大值均值
过去 4 天峰值小时众数
过去 4 天峰谷差
过去 96 小时趋势斜率
同一小时相对历史均值的偏差
```

这些特征把小时级序列压缩为模型更容易学习的结构化信息。

### 1.2 适用范围

统计方法适合：

| 场景 | 原因 |
| --- | --- |
| 数据量不大 | 不需要大量样本即可计算 |
| 周期性较明显 | 可以提取周期均值、同小时差值、峰谷差 |
| 需要解释性 | 每个特征都有明确含义 |
| 深度模型效果差 | 可避免整段序列预测被过度平滑 |

不适合：

| 场景 | 原因 |
| --- | --- |
| 规律极其复杂 | 简单统计量可能表达能力不足 |
| 长期外部因素主导 | 只看历史序列可能信息不够 |
| 峰值随机性很强 | 统计规律难以稳定预测 |

### 1.3 在时序预测中的应用

统计方法常用于：

```text
构造模型输入特征
生成规则基线
检测异常点
提取周期结构
辅助解释模型预测
```

在本实验中，统计方法是第一版主力方法。它用于生成过去 4 天的周期特征、峰值历史特征、趋势特征和差分特征。

## 2. 残差值方法

残差值方法的核心是先建立一个简单可解释的基线预测，再让模型学习真实值相对基线的偏差。

### 2.1 原理

残差定义为：

```text
residual = true_value - baseline_value
```

模型不直接预测 `peak_value`，而是预测：

```text
peak_residual = true_peak_value - baseline_peak
```

最终还原：

```text
pred_peak_value = baseline_peak + pred_peak_residual
```

### 2.2 适用范围

残差方法适合：

| 场景 | 原因 |
| --- | --- |
| 有强基线 | 模型只需要学习修正量 |
| 变量尺度差异大 | 残差比绝对值更稳定 |
| 长预测容易回归均值 | 残差能突出偏离正常水平的部分 |
| 需要判断模型是否有增益 | 可以比较模型残差是否优于预测 0 |

不适合：

| 场景 | 原因 |
| --- | --- |
| 基线很差 | 残差本身会很乱 |
| 残差接近随机 | 模型无法稳定修正 |

### 2.3 在本实验中的作用

本实验中，残差学习是核心策略之一。

第一版做法：

```text
baseline_peak = 过去 4 天 peak_value 的均值或加权均值
target = future_peak_value - baseline_peak
```

LightGBM 和 XGBoost 都预测 `target`，最后加回基线得到最终峰值。

## 3. 小波变换

小波变换是一种时频分析方法。它可以同时观察信号在时间位置和尺度上的变化，适合处理非平稳时间序列。

### 3.1 原理

普通傅里叶变换主要告诉我们“整体有哪些频率”，但不擅长说明“某个频率什么时候出现”。小波变换通过不同尺度的小波函数扫描序列，可以把序列分解为：

```text
低频成分：趋势、缓慢变化
高频成分：突变、噪声、尖峰、局部扰动
```

例如对过去 96 小时序列做小波分解，可以得到趋势项和多个细节项。高频细节项能帮助发现最近是否存在尖峰风险或波动增强。

### 3.2 适用范围

小波变换适合：

| 场景 | 原因 |
| --- | --- |
| 序列非平稳 | 能刻画局部时间变化 |
| 局部尖峰明显 | 高频细节项对尖峰敏感 |
| 需要去噪 | 可保留低频趋势，过滤微小扰动 |
| 需要多尺度特征 | 同时提取短期和长期变化 |

不适合：

| 场景 | 原因 |
| --- | --- |
| 样本很短 | 分解层数有限，特征不稳定 |
| 噪声和真实尖峰难区分 | 可能把有效尖峰当作噪声 |
| 第一版尚未跑通 | 会增加复杂度，难判断贡献 |

### 3.3 在时序预测中的应用

小波可用于：

```text
去噪后再提取峰值
提取高频能量作为波动风险特征
提取低频趋势作为状态特征
捕捉局部突变
辅助拐点检测
```

在本实验中，小波不作为第一版主方法。第二版才加入小波特征，与第一版无小波结果做消融对比。

## 4. LightGBM 与 XGBoost

LightGBM 和 XGBoost 都属于梯度提升树模型。它们通过多个决策树逐步修正前一轮预测误差，适合表格特征、小中等规模数据和非线性关系。

### 4.1 为什么适合本实验

本实验每日样本数量有限，深度学习模型容易过拟合或输出平滑结果。树模型更适合：

```text
统计特征
残差目标
非线性规则
小样本表格数据
特征重要性分析
```

### 4.2 与深度学习的区别

深度学习常直接输入长序列，优化整体预测误差。如果目标是完整未来曲线，模型很容易输出平滑值，从而低估峰值。

本实验改为：

```text
直接预测未来每日 peak_value 和 peak_hour
不先预测完整 336 小时曲线
```

这种任务形式更适合树模型。

## 5. 方法组合方式

本实验采用分阶段组合：

```text
第一版：统计方法 + 规则基线 + 残差学习 + LightGBM/XGBoost
第二版：在第一版基础上加入小波特征
后续：根据结果调整 seq_len、pred_len、模型粒度
```

第一版先验证任务是否有可学习信号。若第一版无法超过规则基线，不应急着加入复杂方法，而应检查数据构造、标签、切分、基线和特征。
