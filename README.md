## 2. 已确认实验目标

本实验目标是处理 `ETTh1.csv` 数据集，以自然日 `00:00-23:00` 为一个周期，预测未来多个自然日内 7 个变量的每日波峰。

目标变量：

```text
HUFL, HULL, MUFL, MULL, LUFL, LULL, OT
```

每个变量每天预测两个目标：

```text
peak_value：当天最大值
peak_hour：当天最大值第一次出现的小时，范围 0-23
```

第一版默认实验配置：

```text
seq_len = 96 小时，即过去 4 天
pred_len = 336 小时，即未来 14 天
周期 = 固定自然日 00:00-23:00
输入 = 7 列历史数据共同作为特征
输出 = 每列单独建模
优先级 = peak_value 和 peak_hour 都重要，但冲突时优先保证 peak_value
```

## 3. 已确认关键决策

| 决策项 | 当前决定 |
| --- | --- |
| 周期切分 | 固定自然日周期，`00:00-23:00` |
| 不完整日期 | 直接丢弃，不作为输入日，也不作为预测目标日 |
| 重复最大值 | 取第一次出现的小时作为 `peak_hour` |
| 数据切分 | 使用 ETTH1 常见 `12个月 / 4个月 / 4个月` 时间切分 |
| 输入变量 | 每个目标都使用 7 列历史作为输入 |
| 输出方式 | 按变量分开建模 |
| 第一版方法 | 规则基线加残差学习，加 LightGBM 和 XGBoost |
| 第一版是否用小波 | 不使用 |
| 第二版方法 | 在第一版基础上加入小波特征 |
| 效果不好时 | 先增加 `seq_len`，再缩短 `pred_len`，最后加入或扩展小波特征 |

## 4. 规则基线模型说明

规则基线是不使用机器学习模型的简单预测规则，作用是给后续 LightGBM、XGBoost 和小波增强实验提供最低对照。后续模型只有稳定超过这些基线，才能说明模型真正学到了额外规律。

当前规则基线分为两类：一类预测每日波峰值 `peak_value`，另一类预测每日波峰出现小时 `peak_hour`。所有基线都只使用训练集统计量或预测起点之前的历史信息，不使用未来真实值。

### 4.1 peak_value 基线

| 基线名称 | 中文说明 | 使用信息 | 作用 |
| --- | --- | --- | --- |
| `mean_last_4` | 过去 4 天峰值均值 | 当前样本预测起点前 4 个完整自然日的峰值 | 捕捉短期峰值水平，是当前最稳的整体峰值基线 |
| `weighted_mean_last_4` | 过去 4 天峰值加权均值 | 当前样本预测起点前 4 个完整自然日的峰值，越近权重越高 | 检查“越近越重要”的短期惯性是否成立 |
| `cycle_mod_4` | 4 天循环位置复用 | 根据 `horizon mod 4` 复用过去 4 天中对应位置的峰值 | 检查 4 天输入窗口内是否存在可复用的日序模式 |
| `weekday_horizon_mean` | 星期与预测天数均值 | 只在训练集内按目标变量、星期、预测天数统计平均峰值，并带有回退策略 | 捕捉训练集中的日历规律和远期 horizon 平均水平 |

`peak_value` 最佳基线选择规则：

```text
按 目标变量 + 预测天数 分组，只使用验证集 target_variable_horizon 层级指标，
选择 MAE 最低的基线。
```

当前验证集最佳基线分布：

```text
weekday_horizon_mean：48 组
mean_last_4：45 组
cycle_mod_4：5 组
weighted_mean_last_4：0 组
```

这说明短期历史均值和训练集日历均值都很重要；但加权均值没有稳定超过简单均值，最近一天的波动可能会带来额外噪声。

### 4.2 peak_hour 基线

| 基线名称 | 中文说明 | 使用信息 | 作用 |
| --- | --- | --- | --- |
| `mode_last_4` | 过去 4 天峰值小时众数 | 当前样本预测起点前 4 个完整自然日的峰值小时 | 捕捉峰值出现时间的短期重复性 |
| `median_last_4` | 过去 4 天峰值小时中位数 | 当前样本预测起点前 4 个完整自然日的峰值小时 | 提供比众数更平滑的时间位置估计 |
| `global_mode` | 训练集变量全局众数 | 训练集内同一目标变量最常见峰值小时 | 捕捉变量自身稳定的峰值时间偏好 |
| `weekday_mode` | 训练集星期众数 | 训练集内同一目标变量、同一星期最常见峰值小时 | 捕捉峰值时间的星期规律 |

`peak_hour` 最佳基线选择规则：

```text
按 目标变量 + 预测天数 分组，只使用验证集 target_variable_horizon 层级指标，
优先选择 ±2h 命中率最高的基线；
如果并列，再选择普通小时误差更低的基线。
```

当前验证集最佳基线分布：

```text
global_mode：28 组
median_last_4：27 组
mode_last_4：26 组
weekday_mode：17 组
```

这说明峰值小时比峰值数值更难预测，不同变量和不同 horizon 适合的简单规则差异较大。后续模型评估时，`peak_hour` 至少要在 ±2h 命中率或环形小时误差上超过这些时间基线，才说明模型有额外价值。

## 7. 如何判断模型是否真正学到东西

模型不能只看训练集误差。首要判断标准是测试集是否稳定超过规则基线。

一个模型通过第一版验证，需要尽量满足：

| 判断项 | 通过标准 |
| --- | --- |
| peak_value 测试集误差 | 优于最佳规则基线 |
| 相对提升 | MAE 稳定降低 5% 以上更有价值 |
| 变量覆盖 | 多数变量提升，而不是只靠单个变量拉低平均误差 |
| horizon 覆盖 | 多数未来天数提升，而不是只在第 1 天有效 |
| 残差信号 | 预测残差与真实残差有正相关 |
| 分布检查 | 预测峰值分布不能明显塌缩 |
| peak_hour | 至少在 ±2h 命中率上优于时间基线 |

如果 `peak_value` 明显提升但 `peak_hour` 一般，模型仍可保留。若 `peak_hour` 提升但 `peak_value` 变差，不作为主模型。
